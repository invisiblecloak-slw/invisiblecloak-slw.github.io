0031-next permutation

Xiu Di / 2021-04-19

问题描述

实现获取下一个排列的函数：将给定数字序列重新排列成字典序中下一个更大的排列。

如果不存在下一个更大的排列，则将数组重新排列盛最小的序列（即升序排列）。

考虑原地修改，且只使用额外常数空间。

示例1

输入: nums = [1,2,3]

输出: [1,3,2]

示例2

输入: nums = [3,2,1]

输出: [1,2,3]

示例3

输入: nums = [1,1,5]

输出: [1,5,1]

示例4

输入: nums = [1]

输出: [1]

提示:

  • 1 <= nums.length <= 100
  • 0 <= nums[i] <= 100

代码

pre = post = -1
for pos in range(len(nums) - 1, 0, -1):
    if nums[pos] > nums[pos - 1]:
        pre, post = pos - 1, pos
        break
    if post == -1:
        nums.reverse()
        return
    for pos in range(len(nums) - 1, pre, -1):
        if nums[pos] > nums[pre]:
            post = pos
            break
    nums[pre], nums[post] = nums[post], nums[pre]
    l, r = pre + 1, len(nums) - 1
    while l < r:
        nums[l], nums[r] = nums[r], nums[l]
        l += 1
        r -= 1

分析

考虑当前数字序列到下一个数字序列的转移：

  • 对于转移到更大数字序列的情况：
    • 发生了一个顺序数对的交换。
    • 由于是向最邻近的下一个数字序列转移，故交换的是靠最右侧的顺序对。
      • 考虑数字序列中的所有顺序对(pre, post)，pre和post代表下标。
      • 若所有顺序对的pre值不同，则最右侧的顺序对为pre值最大的顺序对。
      • 若最大的pre值对应多个post值，则最右侧的顺序对为post值最大的顺序对。
  • 对于不存在下一个更大的排列的情况：
    • 当前数字序列中不存在顺序对。
  • 援引Wikipedia中的算法描述：
    • The following algorithm generates the next permutation lexicographically after a given permutation. It changes the given permutation in-place.
        1. Find the largest index k such that a[k] < a[k+1]. If no such index exists, the permutation is the last permutation.
        1. Find the largest index l greater than k such that a[k] < a[l].
        1. Swap the value of a[k] with that of a[l].
        1. Reverse the sequence from a[k+1] up to and including the final element a[n].
    • 步骤1和步骤2，旨在找到当前数字序列最右侧的顺序对(pre, post)。
    • 步骤3将当前数字序列最右侧的顺序对消除。
    • 步骤4将(pre+1, n)中的元素进行翻转。(pre+1, n)对应一个降序排列的序列。将此降序序列转化为顺序排列的序列。
