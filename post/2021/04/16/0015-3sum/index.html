0015-3sum

Xiu Di / 2021-04-16

问题描述

给定包含n个整数的数组nums。

判断nums中是否存在三个元素a, b, c，使得a + b + c = 0？

找出所有和为0且不重复的三元组。

示例1

输入: nums = [-1, 0, 1, 2, -1, -4]

输出: [[-1, -1, 2], [-1, 0, 1]]

示例2

输入: nums = []

输出: []

示例3

输入: nums = [0]

输出: []

提示:

  • 0 <= nums.length <= 3000
  • -105 <= nums[i] <= 105

注意: 答案中不包含重复的三元组。

代码

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        res = []
        if not nums or len(nums) <= 2: return res
        nums.sort()
        for i in range(len(nums)):
            if nums[i] > 0: return res
            if i > 0 and nums[i] == nums[i-1]: continue
            j, k = i + 1, len(nums) - 1
            while j < k:
                if nums[i] + nums[j] + nums[k] < 0: j += 1
                elif nums[i] + nums[j] + nums[k] > 0: j -= 1
                else:
                    res.append([nums[i], nums[j], nums[k]])
                    while j < k and nums[j] == nums[j + 1]: j += 1
                    while j < k and nums[k] == nums[k - 1]: k -= 1
                    j += 1
                    k -= 1
        return res

分析

  • 考虑特例：当数组为空或数组元素数目小于3时，返回空列表。

  • 对于泛例：在排序后的数组上按从左到右的顺序搜索符合要求的三元组。固定三元组的最小元素后，采用双指针向中间逼近的方法寻找可行解。

    • 若nums[i] > 0，数组的非递减性决定后续不会搜索到可行解。返回当前的可行解列表。

  • 当讨论某最小元素完毕最后，下次讨论的最小元素与这一最小元素不同。保证了可行解的不重复性。

  • 当双指针找到了某一可行解后，两指针从不同值的下标开始下一次讨论。保证可行解的不重复性。

    • 若nums[i] + nums[j] + nums[k] > 0，说明nums[k]过大，将指针k左移。
    • 若nums[i] + nums[j] + nums[k] < 0，说明nums[j]过小，将指针j右移。

  • 时间复杂度: O(n2)。

    • 数组排序O(nlogn)。
    • 指针i遍历数组O(n)，固定i后双指针遍历数组O(n)。
    • 总时间复杂度O(nlogn) + O(n) * O(n) = O(n2)。
